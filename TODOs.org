* Vision
*Sheaf* — a data-oriented configuration system for Hammerspoon automation.

Users configure their automation using orthogonal, composable concepts:
- *Events* - facts about something that happened, organized in a kind hierarchy
- *Commands* - named actions that can be invoked on a component
- *Components* - context-blind units bundling sources + commands + state + config
- *Behaviors* - component-blind rules mapping events to commands via a handler function
- *Tags* - contextual labels attached to component instances, inherited through instance tree
- *Subscriptions* - the only place components, behaviors, and tags meet; the wiring that creates a system

See DESIGN.md for the full design reference.

The system should be:
- *Declarative* - configuration as data, not imperative code
- *Visualizable* - can see what's configured and what's happening
- *Debuggable* - can trace why something did or didn't trigger
- *Queryable* - can inspect state (via SQLite, REPL, UI)

** PROJ Virtual Workspaces [0/1]
Replace native macOS Spaces with a reliable off-screen workspace layer.
Native Spaces have no public API; every private API workaround breaks on each macOS release.

*** LATER [#A] Design virtual workspace system (AeroSpace-style) :architecture:
     - macOS Spaces API is private, broken on Sequoia, and has broken on every 1-2 releases historically
     - AeroSpace approach: inactive workspace windows moved off-screen via hs.window:setFrame()
     - Uses only public Accessibility API (AXPosition/AXSize) — stable, no SIP required
     - Key design questions:
       - [ ] Off-screen coordinates strategy (e.g. x = totalScreenWidth + offset)
       - [ ] How to handle minimized / hidden windows
       - [ ] Mission Control / cmd+tab visibility of off-screen windows
       - [ ] Integration with paper-wm (which currently relies on native spaces for move-to-space)
       - [ ] Workspace ↔ monitor assignment model (one workspace per monitor, or floating?)
       - [ ] Sheaf integration: workspace switches as events, commands to move windows
     - Reference: AeroSpace source (nikitabobko/AeroSpace), FlashSpace (app-level hide/show variant)
